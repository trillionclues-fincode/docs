---
title: 'History Data'
openapi: 'POST /transactionmanagement/history-data'
description: 'Retrieves a paginated and filtered list of transactions for the authenticated customer or a customer managed by the agent/manager.'
---

This endpoint fetches the transaction history, allowing for detailed filtering by amount range, date range, and pagination. It uses a `POST` method because the complex filtering parameters are sent in the request body.

## Request Headers

This administrative endpoint requires a valid **Access Token** for authorization.

<ParamField header='X-Auth-Token' type='string' required>
  The **JWT Access Token** of a user with appropriate permissions (e.g.,
  `CUSTOMER` or `CASHIER`).
</ParamField>

<ParamField header='platform' type='string' required default='fincode'>
        Platform identifier. Use `fincode`.
</ParamField>

<ParamField header='uuid' type='string' required default='200'>
        Unique request identifier.
</ParamField>

<ParamField
  header='Content-Type'
  type='string'
  required
  default='application/json'
>
        Must be `application/json`.
</ParamField>

## Request Body

The request body is used to define the filtering and pagination parameters.

<ParamField body='fromAmount' type='number' required default='0.1'>
  The **minimum** transaction amount to include in the search.
</ParamField>

<ParamField body='toAmount' type='number' required default='100000000'>
  The **maximum** transaction amount to include in the search.
</ParamField>

<ParamField body='fromDate' type='string' required default='YYYY-MM-DD'>
  The start date for the transaction search range (inclusive). Example:
  `2025-09-21`.
</ParamField>

<ParamField body='toDate' type='string' required default='YYYY-MM-DD'>
  The end date for the transaction search range (inclusive). Example:
  `2025-11-20`.
</ParamField>

<ParamField body='startNumber' type='number' required default='1'>
  The index of the first record to return (for pagination).
</ParamField>

<ParamField body='totalNumber' type='number' required default='200'>
  The maximum number of records to return in this request (page size).
</ParamField>

<ParamField body='customerCode' type='string'>
  Optional: The unique customer code to search transactions for. If omitted,
  returns transactions for the authenticated user.
</ParamField>

## Code Examples

<CodeGroup>

```bash cURL
curl -X GET 'https://remitjunction.fincode.software/api/v6/services/transactionmanagement/history-data' \
--header 'X-Auth-Token: YOUR_JWT_ACCESS_TOKEN' \
--header 'platform: fincode' \
--header 'uuid: 200' \
--header 'Content-Type: application/json' \
--data '{

    "toAmount": 100000000,
    "fromDate": "2025-09-21",
    "toDate": "2025-11-20",
    "totalNumber": 200,
    "startNumber": 1
}'
```

```javascript Node.js
const axios = require('axios');

const BASE_URL =
  'https://remitjunction.fincode.software/api/v6/services/transactionmanagement';

const historyPayload = {
  fromAmount: 0.1,
  toAmount: 100000000,
  fromDate: '2025-09-21',
  toDate: '2025-11-20',
  totalNumber: 20,
  startNumber: 1,
};

async function getTransactionHistory(accessToken, payload) {
  try {
    const response = await axios.post(`${BASE_URL}/history-data`, payload, {
      headers: {
        'X-Auth-Token': accessToken,
        platform: 'fincode',
        uuid: '200',
        'Content-Type': 'application/json',
      },
    });

    console.log(`Total Transactions Found: ${response.data.totalCount}`);
    console.log(
      `Displaying ${response.data.transactions.length} records (Page 1):`
    );
    response.data.transactions.forEach((tx) => {
      console.log(
        `- ID: ${tx.transactionId}, Amount: ${tx.amount} ${tx.currency}, Status: ${tx.status}`
      );
    });

    return response.data;
  } catch (error) {
    console.error(
      'Failed to fetch history:',
      error.response?.data || error.message
    );
    throw error;
  }
}

```
