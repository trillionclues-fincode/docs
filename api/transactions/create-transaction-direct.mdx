# Transaction Only Flow

Initiates a new transaction where you provide the full KYC details for both the sender and the recipient within the transaction payload.

This endpoint is ideal for integrations where you manage your own customer data and do not wish to sync it beforehand using the User Management APIs.

## Request Headers

<ParamField header='X-Auth-Token' type='string' required>
  The **JWT Access Token** obtained from the `/login` or `/refresh-token`
  endpoint.
</ParamField>

<ParamField
  header='Content-Type'
  type='string'
  required
  default='application/json'
>
      Must be `application/json`
</ParamField>

<ParamField header='platform' type='string' required default='fincode'>
      Platform identifier. Use `fincode`.
</ParamField>

<ParamField header='uuid' type='string' required default='200'>
      Unique request identifier.
</ParamField>

<ParamField header='password' type='string' required>
  The user's current password. Used for second-factor transaction authorization.
</ParamField>

<ParamField header='ipAddress' type='string' required>
  The originating IP address of the user initiating the transaction.
</ParamField>

## Request Body

<ParamField body='amount' type='string' required>
  The **receiving** amount (including fees if `inclusive` is true). Max 3
  decimal places.
</ParamField>

<ParamField body='sendAmount' type='string' required>
  The **sending** amount. Max 3 decimal places.
</ParamField>

<ParamField body='inclussive' type='boolean' required>
  If `true`, fees are deducted from the `sendAmount`. If `false`, fees are added
  to the `sendAmount`.
</ParamField>

<ParamField body='receivingCurrency' type='string' required default='NGN'>
  The currency the recipient will receive (ISO 4217 code).
</ParamField>

<ParamField body='sendCurrency' type='string' required default='GBP'>
  The currency the sender is paying with (ISO 4217 code).
</ParamField>

<ParamField body='type' type='string' required default='ACCOUNTPAYMENT'>
  Type of transfer being executed. Example: `ACCOUNTPAYMENT`.
</ParamField>

<ParamField
  body='transactionProcessingMode'
  type='string'
  default='NONE_AGGREGATION'
>
  Specifies the aggregation strategy. Example: `NONE_AGGREGATION`.
</ParamField>

<ParamField body='descriptionText' type='string'>
  Optional description for the transfer.
</ParamField>

<ParamField
  body='transactionModule'
  type='string'
  required
  default='REMITTANCE'
>
  The core business module this transaction belongs to. Must be `REMITTANCE`.
</ParamField>

<ParamField body='creditParty' type='array' required>
  Recipient banking/payout details.
  <Expandable title='creditParty object array'>
    <ParamField body='creditParty[].key' type='string' required>
      Key identifier for the detail (e.g., `bankaccountno`).
    </ParamField>
    <ParamField body='creditParty[].value' type='string' required>
      The actual value for the key (e.g., `8883968718`).
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body='internationalTransferInformation' type='object' required>
  Details related to the cross-border nature of the transfer.
  <Expandable title='internationalTransferInformation object'>
    <ParamField
      body='internationalTransferInformation.originCountry'
      type='string'
      required
      default='GBR'
    >
      Country of the sender (ISO 3166-1 alpha-3 code).
    </ParamField>
    <ParamField
      body='internationalTransferInformation.receivingCountry'
      type='string'
      required
      default='NGA'
    >
      Country of the recipient (ISO 3166-1 alpha-3 code).
    </ParamField>
    <ParamField
      body='internationalTransferInformation.sourceOfFunds'
      type='string'
      required
      default='INHERITANCE'
    >
      The source of the money being sent (e.g., `BORROWED_FUNDS/LOAN`, `PENSIONS/GOVERNMENT/WELFARE`, `SALARY`, `INHERITANCE`, `MEDICAL_AID`, `OTHER`).
    </ParamField>
    <ParamField
      body='internationalTransferInformation.remittancePurpose'
      type='string'
      required
      default='EDUCATIONAL_EXPENSES'
    >
      The purpose of the transfer (e.g., `FAMILY_SUPPORT`, `EDUCATIONAL_EXPENSES` `PAYBACK`, `OTHER`,
      `FAMILY_SUPPORT`).
    </ParamField>
    <ParamField
      body='internationalTransferInformation.relationshipSender'
      type='string'
    >
      Relationship of the sender to the recipient.
    </ParamField>
    <ParamField
      body='internationalTransferInformation.deliveryMethod'
      type='string'
      required
      default='directtoaccount'
    >
      The method for paying out the funds. Example: `directtoaccount`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body='newSenderKyc' type='object' required>
  Object containing full KYC details for the sender.
   <Expandable title='newSenderKyc object'>
      <ParamField body='nationality' type='string'>Sender's nationality</ParamField>
      <ParamField body='dateOfBirth' type='string'>Sender's date of birth (YYYY-MM-DD)</ParamField>
      <ParamField body='postalAddressObject' type='object'>Sender's address object</ParamField>
   </Expandable>
</ParamField>

<ParamField body='newRecipientKyc' type='object' required>
  Object containing full KYC details for the recipient.
    <Expandable title='newRecipientKyc object'>
        <ParamField body='nationality' type='string'>Recipient's nationality</ParamField>
    </Expandable>
</ParamField>
