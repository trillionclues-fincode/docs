---
title: 'Awaiting Compliance'
api: 'GET https://remitjunction.fincode.software/api/v6/services/compliance/awaiting-compliance-requirements'
description: 'Checks for any outstanding Know Your Customer (KYC) or regulatory documents required from the authenticated user.'
---

This endpoint is used to quickly check if the authenticated user has any pending or mandatory compliance actions or documents required before proceeding with certain high-risk actions (like large transactions or profile changes).

<RequestExample>
          ```bash cURL GET        
  [https://remitjunction.fincode.software/api/v6/services/compliance/awaiting-compliance-requirements](remitjunction://agent.fincode.software/api/v6/services/compliance/awaiting-compliance-requirements)
          ```
</RequestExample>

## Request Headers

This endpoint requires a valid **Access Token** for authorization.

<ParamField header='X-Auth-Token' type='string' required>
  The **JWT Access Token** obtained from the `/login` or `/refresh-token`
  endpoint.
</ParamField>

<ParamField header='platform' type='string' required default='fincode'>
        Platform identifier. Use `fincode`.
</ParamField>

<ParamField header='uuid' type='string' required default='200'>
        Unique request identifier.
</ParamField>

## Response

The response indicates the overall status of compliance and details any pending actions.

<ResponseField name='status' type='string' default='SUCCESS'>
  Overall status of the API request.
</ResponseField>

<ResponseField name='message' type='string'>
  A descriptive message about the compliance status for the user, e.g., "No
  documents required from [User Name]".
</ResponseField>

<ResponseField name='data' type='object'>
  Container for the specific compliance data.
  <Expandable title='data object'>
    <ResponseField name='awaitingComplianceActions' type='array'>
      A list of specific documents or actions the user is required to complete.
      This array is empty if no documents are required.
      <Expandable title='awaitingComplianceActions item'>
        <ResponseField name='documentType' type='string'>
          The type of document required (e.g., `PROOF_OF_ADDRESS`,
          `ID_VERIFICATION`).
        </ResponseField>
        <ResponseField name='reason' type='string'>
          The compliance reason for the requirement (e.g., `ANNUAL_REVIEW`,
          `TRANSACTION_THRESHOLD`).
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name='status' type='string' default='Success'>
      The compliance check status.
    </ResponseField>
  </Expandable>
</ResponseField>

## Code Examples

<CodeGroup>

```bash cURL
curl --location 'https://remitjunction.fincode.software/api/v6/services/compliance/awaiting-compliance-requirements' \
--header 'X-Auth-Token: YOUR_JWT_ACCESS_TOKEN' \
--header 'platform: fincode' \
--header 'uuid: 200'
```

```javascript Node.js
const axios = require('axios');

const BASE_URL = 'https://agent.fincode.software/api/v6/services/compliance';

async function checkComplianceRequirements(accessToken) {
  try {
    const response = await axios.get(
      `${BASE_URL}/awaiting-compliance-requirements`,
      {
        headers: {
          'X-Auth-Token': accessToken,
          platform: 'fincode',
          uuid: '200',
        },
      }
    );

    const { awaitingComplianceActions, status } = response.data.data;

    console.log(`Compliance Check Status: ${status}`);
    if (awaitingComplianceActions.length > 0) {
      console.log(
        `PENDING ACTIONS: ${awaitingComplianceActions.length} documents required.`
      );
      awaitingComplianceActions.forEach((action) => {
        console.log(
          `- Type: ${action.documentType} (Reason: ${action.reason})`
        );
      });
    } else {
      console.log('User is fully compliant. No documents required.');
    }

    return response.data;
  } catch (error) {
    console.error(
      'Compliance check failed:',
      error.response?.data || error.message
    );
    throw error;
  }
}

// Usage Example
const accessToken = 'YOUR_JWT_ACCESS_TOKEN';
checkComplianceRequirements(accessToken);
```
