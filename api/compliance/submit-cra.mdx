---
title: 'Submit CRA Form'
openapi: 'POST /admin/compliance/submit-cra'
description: 'Submits the data collected from a dynamic form, such as a Customer Risk Assessment (CRA), for processing and compliance evaluation.'
---

This endpoint is used to submit the collected data from dynamic forms (e.g., CRA or KYC forms). The system validates the input, updates the customer's compliance status, and may automatically trigger the next step in the enrolment process (like external screening).

## Request Headers

This compliance/administrative endpoint requires a valid **Access Token** for authorization.

<ParamField header='X-Auth-Token' type='string' required>
  The **JWT Access Token** of the user submitting the form.
</ParamField>

<ParamField header='platform' type='string' required default='fincode'>
        Platform identifier. Use `fincode`.
</ParamField>

<ParamField header='uuid' type='string' required default='200'>
        Unique request identifier.
</ParamField>

<ParamField
  header='Content-Type'
  type='string'
  required
  default='application/json'
>
        Must be `application/json`.
</ParamField>

## Request Body

The request body contains the identifiers of the form and the customer, along with the collected data.

<ParamField
  body='dynamicFormId'
  type='string'
  required
  default='31564b03-f183-44e3-ae4a-fa343c6f7e3a'
>
  The unique ID of the specific dynamic form template being submitted.
</ParamField>

<ParamField
  body='customerId'
  type='string'
  required
  default='daa02d5e-2be1-4523-ba6e-1930045b0e53'
>
  The unique ID of the customer the form data belongs to.
</ParamField>

<ParamField body='dynamicFieldIds' type='object' required>
  A map of field IDs to the corresponding user-submitted values.
  <Expandable title='dynamicFieldIds map'>
    <ParamField body='dynamicFieldIds.{fieldId}' type='string' required>
      Key is the **field ID** (UUID) from the dynamic form definition, and the
      value is the **data** entered by the user.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body='kycRuleId' type='string'>
  Optional ID of a specific KYC rule that mandated this form submission (can be
  `null`).
</ParamField>

```javascript Node.js
const axios = require('axios');

const BASE_URL =
  'https://remitjunction.fincode.software/api/v6/services/admin/compliance';

const submissionPayload = {
  dynamicFieldIds: {
    '3b1a2bea-ce96-4110-8360-9dc4c686119a': 'Male',
    'f37a8f93-89a4-4fbf-a21d-e36643922d5a': 'GBR',
    '6b103f8d-84ca-4313-b68e-e0a0d1b3bc52': '£1,501 – £3,000',
    // ... include all required fields
  },
  dynamicFormId: '31564b03-f183-44e3-ae4a-fa343c6f7e3a',
  customerId: 'daa02d5e-2be1-4523-ba6e-1930045b0e53',
  kycRuleId: null,
};

async function submitCRA(accessToken, payload) {
  try {
    const response = await axios.post(`${BASE_URL}/submit-cra`, payload, {
      headers: {
        'X-Auth-Token': accessToken,
        platform: 'fincode',
        uuid: '200',
        'Content-Type': 'application/json',
      },
    });

    console.log(`CRA Submission Status: ${response.data.message}`);
    console.log(`New Risk Level: ${response.data.data.newCustomerRiskLevel}`);
    console.log(`Next Action: ${response.data.data.nextStep}`);

    return response.data;
  } catch (error) {
    console.error(
      'CRA submission failed:',
      error.response?.data || error.message
    );
    throw error;
  }
}

// Usage Example
const accessToken = 'YOUR_JWT_ACCESS_TOKEN';
submitCRA(accessToken, submissionPayload);
```
